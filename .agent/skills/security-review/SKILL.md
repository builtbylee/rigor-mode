# Skill: security-review

## Goal
Prevent common security failures in everyday engineering work.

## Checklist (minimum)
- Secrets: no API keys/tokens committed
- Auth: verify authn/authz boundaries
- Input handling: validate/sanitize user inputs
- Injection risks: SQL, command, template
- Logging: avoid sensitive data in logs
- Dependencies: avoid untrusted execution paths
